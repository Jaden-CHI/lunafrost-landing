01
HERO · ABOVE THE FOLD

정적인 명함이 아니라,
입구가 되어야 합니다.

현재 Hero는 "이름 + 한 줄 설명 + 두 버튼"의 명함 레이아웃입니다. 풀스크린 높이를 차지하면서도 정보 밀도는 명함 한 장 분량이고, 시각적 앵커가 타이포 외엔 없습니다. Cold Explorer 톤이라면 더 절제하더라도 더 "기록자의 작업실 입구" 같아야 합니다.

ISSUE 1.1 HIGH

두 개의 CTA가 경쟁합니다.

"블로그 보기"는 채워진 파란 버튼(#8fb8dc), "ABOUT"은 outline 다크 버튼. 둘 다 비슷한 크기, 비슷한 무게, 좌우 나란히 정렬되어 시선을 분산시킵니다. 게다가 한국어 "블로그 보기"는 lowercase tracking, "ABOUT"은 uppercase로 처리되어 톤이 어긋납니다.

DM.md: ghost primary + glow 현재: solid pill, no glow
RECOMMENDATION
  • Primary CTA는 ghost + rim-light로 강화 — bloom 효과로 "frosted" 느낌 살리기 (DESIGN.md 컴포넌트 가이드 그대로)
  • Secondary는 단순 텍스트 + underline으로 격하 — "Read About →" 형태로 inline 처리
  • 두 CTA의 패딩·폰트사이즈에 위계 차이를 명확히 둘 것 (primary 1.2x)
  • 한·영 혼용 대신 "블로그 보기 / 소개 →" 또는 "Read the Blog / About"로 통일

ISSUE 1.2 HIGH

화면의 80%가 비어 있습니다.

중앙 정렬된 텍스트 블록을 빼면 위·아래·좌우 전부 공백입니다. 디자인 시스템이 약속한 "razor-thin grid lines"는 background-size 60px / opacity 0.03으로 사실상 보이지 않습니다. 잡지 표지였다면 부제, 메타데이터, 챕터 인덱스가 모서리에 배치되어 화면을 채웠을 것입니다.

RECOMMENDATION

  • 우측 하단: 위치/시간 (SEOUL · 14:22 KST) — 기록자의 페르소나 강화
  • 좌측 상단: 챕터 인덱스 (VOL.05 / 2026) — 잡지 메타포
  • 그리드 라인 opacity를 0.05~0.08로 올리고, 60px → 80px로 spacing 키워 시각적 무게감 부여
  • "AI · DEV · APP · CONTENTS" 케이스: 단순 중앙 텍스트 대신, 각 단어를 화면 모서리에 분산 배치하는 editorial 트릭 고려

ISSUE 1.3 MED

"lunafrost" 타이포 트리트먼트가 약합니다.

"frost" 부분을 font-extralight + text-primary로 처리한 의도는 좋지만, 두 단어의 무게 차이가 너무 미묘해서 디바이스/렌더링에 따라 "그냥 좀 흐릿한 글자"로 읽힙니다. 브랜드 이름이 핵심 시각 자산이라면 더 명확한 시각적 장치가 필요합니다.

옵션 — 셋 중 하나 선택
  • A. Weight contrast 강화 — "luna" 400 italic + "frost" 300 italic, 자간 차이 추가
  • B. Stroke vs Fill — "luna"는 outline (text-stroke), "frost"는 fill — 진짜 frost 메타포
  • C. 글자 사이 separator — "luna · frost" 또는 "luna / frost"로 명시적 분리 + 위에 작은 latitude 좌표 같은 코드

ISSUE 2.2 MED

Status pill의 색 사용이 일관되지 않습니다.

"운영 중", "출시 완료", "개발 중" — 세 가지 상태가 있지만 그 중 하나만 primary 컬러로 강조됩니다. 디자인 시스템은 색을 의미 단위로 써야 합니다.

RECOMMENDATION
  • 상태별 컬러 매핑을 명시적으로 정의 — 예: 운영 중 = green dot, 출시 완료 = primary, 개발 중 = warn(amber), 일시정지 = mute
  • Pill 대신 "색 도트 + 텍스트" 패턴 — 더 절제되고 기술적 — DESIGN.md의 "ledger" 톤과 맞음
  • 또는 status를 카드 모서리의 미세한 인디케이터 라인으로 — 좌측 1px 컬러 라인 (left-border 트로프 NO, 짧은 horizontal rule로)

ISSUE 2.3 MED

"자세히 보기" 버튼이 카드 무게의 50%를 차지합니다.

풀 와이드 보더 버튼은 카드 하단을 무겁게 만듭니다. 위계상 가장 약해야 할 요소가 가장 큰 면적을 차지하는 모순. 기술 태그(chip)와 충돌하기도 합니다.

RECOMMENDATION
  • 버튼을 텍스트 링크로 격하 — 자세히 보기 → 우측 정렬, 우측 화살표만 hover에 반응
  • 또는 카드 전체를 클릭 가능하게 만들고, hover시 카드 자체가 인디케이터 — 별도 CTA 제거
  • 현재 hover 상태에서 border가 primary로 바뀌는데, 카드 전체가 활성화되는 표현으로 확장 (예: 좌측에 1px primary 라인이 "그려지듯" 나타남 — DESIGN.md의 "Interactive Lines that grow")

CORE VALUES · STATS

"08+"는 0으로 시작하는
경력이 되어버립니다.

전체 사이트가 한국어인데 통계 라벨만 영어("Years Experience / Projects Shared"), 그리고 leading zero는 8년차를 "0년차에 가깝게" 보이게 만듭니다. 통계 옆에 컨텍스트도 없어서 "그래서 뭐?"라는 질문을 유발합니다.

ISSUE 3.1 MED

통계가 장식으로만 작동합니다.

"08+ Years Experience", "124+ Projects Shared" — 숫자만 크게 박혀 있고 맥락이 없습니다. 이력 페이지가 따로 있다는 신호도 없고, 클릭 가능하지도 않습니다.

RECOMMENDATION
  • "08+" → 8 또는 8.5 — 정직한 숫자로
  • 라벨을 한국어로 통일 — "년차 / 공유한 프로젝트" 또는 mono로 영문 (단, 화면 어디에서나 일관)
  • 3~4개로 늘려서 통계 면적을 시각 자산으로 — 예: "8년차 · 124 프로젝트 · 8.4M 누적 조회 · 42 글"
  • 각 숫자 옆에 미세한 trend chart (마지막 12개월) — DESIGN.md의 "data visualization" 약속 이행
  • 숫자에 hyperlink — 각각 해당 페이지로 (포트폴리오, 블로그, YouTube)

01 / 06

의도적 Misalignment

"Elements are often offset or intentionally misaligned with the grid to create an editorial 'magazine' feel" — 현재 모든 요소가 그리드 정중앙. Hero·Portfolio·Core Values 모두 대칭 레이아웃입니다.

FIX · 한두 섹션을 비대칭으로
02 / 06

Rim-light & Bloom

"active elements use 1px inner-stroke / rim-light to simulate light hitting a frosted surface" — CSS에 .rim-light 클래스가 있지만 거의 보이지 않을 만큼 약함 (0.05 opacity).

FIX · opacity 0.05 → 0.12
03 / 06

Growing Lines

"thin vertical and horizontal lines as separators that appear to grow or animate in as the user scrolls" — 현재 모든 라인이 정적. Scroll-trigger 애니메이션 부재.

FIX · IntersectionObserver + scaleX
04 / 06

Custom Lens Cursor

"a custom circular lens cursor that creates a localized frost-clear effect or slight magnification on portfolio images" — 미구현. 포트폴리오 이미지 hover에서 시그니처 인터랙션이 될 수 있는 디테일.

FIX · CSS mix-blend + backdrop-filter
05 / 06

AI Gradient Stroke

"AI related tags use subtle purple-to-blue gradient stroke" — AI 도구·Claude API 등 AI 태그가 일반 surface-container와 동일하게 보임. 차별화 안 됨.

FIX · border-image gradient on .tag-ai
06 / 06

Terminal-style 인풋

"Input Fields: Bottom-border only by default, evoking a terminal or a ledger" — 현재 인풋 자체가 없음. 뉴스레터·검색·연락폼이 추가될 자리에 미리 디자인 적용 필요.
